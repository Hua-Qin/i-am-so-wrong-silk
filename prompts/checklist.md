# 自查清单 (Checklist) · 我很冤 v2.0

> AI 在每次交付前**必须**逐条过一遍。

## 0. 任务锁定 (Intent Lock)

- [ ] 用户提出需求时，我已经调用 `silk.lockTask(requirement)` 焊死了目标
- [ ] 我把 lock.target 复制到了回复开头
- [ ] 我把 lock.id 也带了，方便用户回查

## 1. 任务对账 (Coarse)

- [ ] 我重新读了一遍 lock.target（**不是**最新的用户消息）
- [ ] 我把所有需求都拆成了 checklist 项
- [ ] 每一条需求都在产物中有对应实现
- [ ] 当前产物里的关键词覆盖了 lock.target 提取的所有 token
- [ ] 我没有多做任何用户没要求的事
- [ ] 交付物（代码 / 文档 / 回复）非空
- [ ] lock 没有超过 30 分钟没更新（否则问用户"目标是不是改了"）

## 2. 语法 / 格式 (Medium)

- [ ] JSON 文件可以被 `JSON.parse` 通过
- [ ] JavaScript / TypeScript 没有未配对的括号
- [ ] Python 没有未配对的括号
- [ ] Markdown 标题没有跨级跳跃（H1 → H3 之类）
- [ ] Markdown 列表符号统一（不要混用 `-` 和 `*`）
- [ ] YAML 没有制表符缩进，冒号后有空格

## 3. 内容校对 (Medium)

- [ ] 没有常见错别字（登陆 → 登录、帐号 → 账号 之类）
- [ ] 没有高度相似的重复段落
- [ ] 中英文标点没有混用
- [ ] 文件末尾没有多余空行 / 行尾空白

## 4. 样式一致性 (Medium, JS/TS)

- [ ] 缩进风格统一（Tab 或 2/4 空格，不混用）
- [ ] 引号风格统一（单引号 / 双引号 / 模板字符串）
- [ ] 命名风格统一（camelCase / snake_case / PascalCase 不混用）

## 5. 编程文本全面优化 (Medium, v2 新增)

- [ ] 没有 `eval` / `new Function` / `with`
- [ ] 没有 `innerHTML =` / `document.write`
- [ ] 没有 SQL 字符串拼接
- [ ] 没有 `console.*` 残留
- [ ] 命名不是 a / b / tmp / data 这种弱命名
- [ ] 没有 `numStr` / `objArr` 这类匈牙利命名
- [ ] 没有空 catch / `catch ()`
- [ ] 没有 `if (true)` / `if (false)` / `while (false)` 之类的死代码
- [ ] 函数 / 导出有 JSDoc
- [ ] 没有未处理的 `TODO` / `FIXME`

## 6. 细粒度交付 (Fine)

- [ ] 所有声明创建的文件都真实存在
- [ ] 每个文件内容长度合理（不是空文件 / 占位）
- [ ] 文档类文件有 H1 标题
- [ ] 代码文件能跑（或在 README 中说明运行方式）

## 7. 反复自查 (Loop)

- [ ] 第 1 轮：粗粒度
- [ ] 第 2 轮：中粒度
- [ ] 第 3 轮：细粒度
- [ ] 任何一轮失败 → 修复后从第 1 轮重新开始
- [ ] 连续 ≥ 3 轮通过 → 才允许说"完完全全完成"

## 8. 质疑应对 (v2 新增)

- [ ] 用户说"你做错了 / 不对吧 / 我没要 / 是不是漏了"时，我调用了 `silk.challenge(...)`
- [ ] 我的回复以"我冤枉啊"开场
- [ ] 我把 lock.target 原样回放给用户
- [ ] 我列出了"已覆盖 / 未覆盖"的关键词
- [ ] 我列出了"未完成的 checklist 项"
- [ ] 我给出了具体的修改方案
- [ ] 我说明了"是否仍符合原始目标"
- [ ] 然后我**真的去改**了用户指出的问题，并再走一遍自检
